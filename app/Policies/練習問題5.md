【仕様書】会員制施設予約管理システム

■ 概要
会員制施設の予約を管理するシステム。
会員種別によって利用ルールが異なり、
予約時に残泊数・有効期限を自動制御する。

■ テーブル設計
テーブル名: members
- id
- code: string(20) 必須 ユニーク
- name: string(255) 必須
- email: string(255) 必須 ユニーク
- member_type: string デフォルト'standard'
- point_balance: integer デフォルト0
- status: string デフォルト'active'
- created_at / updated_at

テーブル名: plans
- id
- name: string(255) 必須
- plan_type: string 必須（fixday / freeday）
- total_nights: integer 必須
- valid_months: integer 必須
- price: integer 必須
- created_at / updated_at

テーブル名: member_plans
- id
- member_id: foreignId（members）必須
- plan_id: foreignId（plans）必須
- start_date: date 必須
- expire_date: date 必須（start_date + valid_months）
- remaining_nights: integer 必須
- status: string デフォルト'active'
- created_at / updated_at

テーブル名: reservations
- id
- member_plan_id: foreignId（member_plans）必須
- check_in: date 必須
- check_out: date 必須
- nights: integer 必須
- room_type: string 必須
- status: string デフォルト'pending'
- note: text nullable
- created_at / updated_at

テーブル名: point_logs
- id
- member_id: foreignId（members）必須
- type: string 必須（add / use）
- amount: integer 必須
- reason: string(255) nullable
- created_at / updated_at

■ 画面一覧
1. 会員一覧・登録・編集・詳細
2. プラン一覧・登録・編集
3. 会員プラン付与画面（/members/{member}/plans/create）
   - プランを選択すると expire_date を自動計算して表示
   - remaining_nightsはplan.total_nightsで初期設定
4. 予約一覧（/member-plans/{memberPlan}/reservations）
   - 該当プランの予約一覧
5. 予約登録画面
   - check_in / check_out を入力するとnightsを自動計算
   - remaining_nightsを超えるnightsはバリデーションエラー
   - 予約登録時にmember_plans.remaining_nightsを減算
   - 予約キャンセル時にremaining_nightsを加算
6. ポイント付与画面（/members/{member}/points）
   - amount / reason を入力
   - 登録後 members.point_balance を加算・減算
   - point_logsに履歴を記録

■ バリデーション（Reservation）
- check_in: required|date|after_or_equal:today
- check_out: required|date|after:check_in
- nights: required|integer|min:1
- room_type: required|in:standard,deluxe,suite
- status: required|in:pending,confirmed,cancelled
- note: nullable|string
- nightsがremaining_nightsを超えていたらエラー

■ Service層要件（必須）
- ReservationService
  - 予約登録時の残泊数減算処理
  - 予約キャンセル時の残泊数加算処理
- PointService
  - ポイント付与・消費処理
  - point_logsへの履歴記録
  - members.point_balanceの更新

■ 命名規則
- Models: Member / Plan / MemberPlan / Reservation / PointLog
- Controllers: MemberController / PlanController / 
               MemberPlanController / ReservationController /
               PointController
- Services: ReservationService / PointService
- ルート:
  Route::resource('members', MemberController::class);
  Route::resource('plans', PlanController::class);
  Route::resource('members.plans', MemberPlanController::class);
  Route::resource('member-plans.reservations', ReservationController::class);
  Route::resource('members.points', PointController::class);
- View: 各リソースに対応したディレクトリ
