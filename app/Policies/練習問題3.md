【仕様書】社員勤怠管理システム

■ 概要
社員の出退勤記録を管理する画面を作成する。
日付・社員ごとの勤務時間を計算して表示する。

■ テーブル設計
テーブル名: employees
- id
- code: string(20) 必須 ユニーク
- name: string(255) 必須
- department: string(100) 必須
- is_active: boolean デフォルトtrue
- created_at / updated_at

テーブル名: attendances
- id
- employee_id: foreignId（employees）必須
- work_date: date 必須
- clock_in: time 必須
- clock_out: time nullable
- break_minutes: integer デフォルト60
- note: text nullable
- created_at / updated_at

■ 画面一覧
1. 社員一覧（/employees）
   - テーブル形式
   - 各行に「勤怠一覧」リンク

2. 社員登録・編集画面

3. 勤怠一覧（/employees/{employee}/attendances）
   - 該当社員の勤怠をwork_date降順で表示
   - 実働時間を表示（clock_outがある場合: clock_out - clock_in - break_minutes）
   - clock_outがnullの場合は「退勤未打刻」と表示

4. 勤怠登録・編集画面
   - work_date / clock_in / clock_out / break_minutes / note を入力

■ バリデーション（Employee）
- code: required|string|max:20|unique:employees（編集時自分除外）
- name: required|string|max:255
- department: required|string|max:100

■ バリデーション（Attendance）
- employee_id: required|exists:employees,id
- work_date: required|date
- clock_in: required|date_format:H:i
- clock_out: nullable|date_format:H:i|after:clock_in
- break_minutes: required|integer|min:0|max:480
- note: nullable|string

■ Controller追加要件
- AttendanceControllerはemployeeにネストしたresourceルート
- 実働時間の計算はモデルのアクセサで実装

■ 命名規則
- Models: Employee / Attendance
- Controllers: EmployeeController / AttendanceController
- Requests: 各モデルにStore / Update
- ルート: 
  Route::resource('employees', EmployeeController::class);
  Route::resource('employees.attendances', AttendanceController::class);
- View: resources/views/employees/ / resources/views/attendances/
