【仕様書】請求・入金管理システム

■ 概要
請求書の発行と入金消込を管理するシステムを作成する。
請求に対して複数の入金を紐づけ、未入金残高を自動計算する。

■ テーブル設計
テーブル名: clients
- id
- name: string(255) 必須
- email: string(255) nullable
- created_at / updated_at

テーブル名: invoices
- id
- client_id: foreignId（clients）必須
- invoice_no: string(50) 必須 ユニーク
- issue_date: date 必須
- due_date: date 必須
- amount: integer 必須
- status: string デフォルト'unpaid'
- note: text nullable
- created_at / updated_at

テーブル名: payments
- id
- invoice_id: foreignId（invoices）必須
- paid_date: date 必須
- amount: integer 必須
- method: string デフォルト'bank_transfer'
- note: text nullable
- created_at / updated_at

■ 画面一覧
1. 請求一覧（/invoices）
   - client名 / invoice_no / amount / 入金済合計 / 未入金残高 / status を表示
   - statusがpaidは緑・unpaidは赤・cancelledはグレー

2. 請求詳細（/invoices/{invoice}）
   - 請求情報を表示
   - 紐づく入金一覧をテーブルで表示
   - 入金済合計・未入金残高を表示
   - 入金登録ボタン

3. 請求登録・編集画面
   - client_idはselectで選択

4. 入金登録画面（/invoices/{invoice}/payments/create）
   - paid_date / amount / method / note を入力
   - methodはselect: bank_transfer / cash / card

■ バリデーション（Invoice）
- client_id: required|exists:clients,id
- invoice_no: required|string|max:50|unique:invoices（編集時自分除外）
- issue_date: required|date
- due_date: required|date|after_or_equal:issue_date
- amount: required|integer|min:1
- status: required|in:unpaid,paid,cancelled
- note: nullable|string

■ バリデーション（Payment）
- invoice_id: required|exists:invoices,id
- paid_date: required|date
- amount: required|integer|min:1
- method: required|in:bank_transfer,cash,card
- note: nullable|string

■ Controller追加要件
- 未入金残高 = invoice.amount - payments.sum('amount')
- 入金登録後、未入金残高が0以下になったら自動でstatus='paid'に更新
- 残高計算はInvoiceモデルのアクセサで実装
- PaymentControllerはinvoiceにネスト

■ 命名規則
- Models: Client / Invoice / Payment
- Controllers: ClientController / InvoiceController / PaymentController
- Requests: 各モデルにStore / Update
- ルート:
  Route::resource('clients', ClientController::class);
  Route::resource('invoices', InvoiceController::class);
  Route::resource('invoices.payments', PaymentController::class);
- View: resources/views/clients/ / invoices/ / payments/
