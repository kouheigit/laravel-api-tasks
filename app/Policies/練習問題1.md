【仕様書】顧客管理システム

■ 概要
営業部門が使う顧客情報の管理画面を作成する。
顧客の登録・閲覧・編集・削除ができること。

■ テーブル設計
テーブル名: customers
- id
- company_name: string(255) 必須
- contact_name: string(255) 必須
- email: string(255) 必須 ユニーク
- phone: string(20) nullable
- status: string デフォルト'active' 
- memo: text nullable
- created_at / updated_at

■ 画面一覧
1. 一覧画面
   - テーブル形式で表示
   - company_name / contact_name / email / status を表示
   - 各行に詳細・編集・削除ボタン
   - 新規登録ボタン

2. 詳細画面
   - 全カラムを表示

3. 新規登録画面
   - 全項目入力フォーム
   - statusはselectで active / inactive

4. 編集画面
   - 既存データを初期値としてフォーム表示

■ バリデーション
- company_name: required|string|max:255
- contact_name: required|string|max:255
- email: required|email|max:255|unique:customers（編集時は自分除外）
- phone: nullable|string|max:20
- status: required|in:active,inactive
- memo: nullable|string

■ その他
- 削除前にconfirmダイアログ
- 登録・更新・削除後にフラッシュメッセージ
- バリデーションエラーを画面表示

■ 命名規則
- Model: Customer
- Controller: CustomerController
- Request: StoreCustomerRequest / UpdateCustomerRequest
- ルート: resource（customers）
- View: resources/views/customers/
